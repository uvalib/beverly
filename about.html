General

Beverly is an indexing system for Fedora digital object repositories and Apache Solr indexes.

Features

Beverly can index multiple repositories, and can use (in the extreme case) a separate indexing configuration for each and every object therein. It has highly-parallelized internal workflows and takes advantage of message queuing to offer good scaling characteristics. The index record object

Properties

Beverly relies on Fedora's ability to publish events about changes to repository contents. The configuration for indexing workflows is held in the repositories right next to the objects being indexed.
